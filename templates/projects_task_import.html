Top-level shape: {"tasks": [...]}. The importer also accepts a few aliases such as project_tasks, items, choices, tests, function_call, and method_call.

Code samples can also set compare_mode. Function/class samples may set tolerance when compare_mode is numeric_tolerance.

Text fields support Markdown. Because this is JSON, write line breaks as \n and blank lines as \n\n inside the string value.

{
  "prompt": "Read the instructions carefully.\\n\\n- Use a loop\\n- Print one result per line",
  "problem_statement": "Starter shape:\\n```python\\ndef solve():\\n    pass\\n```"
}

Quick JSON Tutorial

1. Start with one task

Wrap everything in {"tasks": [...]}. Each task needs a title and a non-empty questions array.

2. Pick the right code mode

Use script for stdin/stdout programs, function for direct calls, and class when each test should build a fresh obj.

3. Add sample comparisons

script samples accept rstrip, exact, normalize_whitespace, and contains. Callable samples accept exact, contains, and numeric_tolerance.

4. Use tolerance only for numeric values

When you use numeric_tolerance, provide a literal numeric expected value or array plus a non-negative tolerance, such as 1e-6.

{
  "type": "code",
  "prompt": "Implement mean(values).",
  "problem_statement": "Return the arithmetic mean of the numbers.",
  "code_mode": "function",
  "function_signature": "def mean(values):",
  "tests": [
    {
      "name": "simple average",
      "function_call": "mean([1, 2, 3])",
      "expected_return": 2.0,
      "compare_mode": "numeric_tolerance",
      "tolerance": 1e-6
    }
  ]
}
