Top-level shape: {"tasks": [...]}. The importer also accepts a few aliases such as project_tasks, items, choices, tests, function_call, and method_call.

Use "task_kind": "tutorial" for read-only lessons. Tutorial tasks should contain markdown in instructions or tutorial_markdown and should not define any questions.

Public code samples belong in tests or samples. Private submit-time judge cases belong in hidden_tests. Hidden tests can use inline stdin/expected_stdout or file-backed stdin_file, expected_file, and mounted files.

Code samples can also set compare_mode. Function/class samples may set tolerance when compare_mode is numeric_tolerance. Hidden tests may also set timeout_ms.

Text fields support Markdown. Because this is JSON, write line breaks as \n and blank lines as \n\n inside the string value.

Upload a private ZIP when imported hidden tests reference stdin_file, expected_file, or mounted files. During import, that ZIP is copied to each imported task that needs it.

Quick JSON Tutorial

1. Start with one task

Wrap everything in {"tasks": [...]}. Each task needs a title. Assessment tasks also need a non-empty questions array.

2. Pick the right code mode

Use script for stdin/stdout programs, function for direct calls, and class when each test should build a fresh obj.

3. Use tutorial tasks for lessons

Set task_kind to tutorial and put the lesson in instructions or tutorial_markdown. Students complete it by opening it, so no questions are needed.

4. Add sample comparisons

script samples accept rstrip, exact, normalize_whitespace, and contains. Callable samples accept exact, contains, and numeric_tolerance.

5. Use hidden tests for final judging

Put Codeforces-style final checks in hidden_tests. Use inline inputs for small private cases and upload a judge ZIP for large file-backed cases. Hidden tests run only on submit, not in the public sample runner.

6. Use tolerance only for numeric values

When you use numeric_tolerance, provide a literal numeric expected value or array plus a non-negative tolerance, such as 1e-6.

{
  "type": "code",
  "prompt": "Implement mean(values).",
  "problem_statement": "Return the arithmetic mean of the numbers.",
  "code_mode": "function",
  "function_signature": "def mean(values):",
  "tests": [
    {
      "name": "simple average",
      "function_call": "mean([1, 2, 3])",
      "expected_return": 2.0,
      "compare_mode": "numeric_tolerance",
      "tolerance": 1e-6
    }
  ]
}
